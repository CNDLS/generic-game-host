LEV/L simplifies game development for educators and university faculty by dividing the labor required to build a game.

  • Faculty focus on the 'script,' expressed in a Game Design Document. This is a highly-structured document, because it needs to be read by a computer, but it is designed to be relatively simple to create and edit. The format is very flexible; you can specify as much or as little complexity as you want. Based on the player's actions, the game can:
    • award or take back points,
    • skip forward or loop back to other rounds,
    • enforce time limits,
    • trigger animations,
    • make resources available to the player,
    • and many other options...
  • CNDLS student assistants and other support staff create scripts and style documents that describe the look and feel of the game. The game is just a Web page where elements are added or removed to create scenes where the game takes place. You can style these with ordinary Web stylesheets. By following a simple structure, the game scripts can both respond to and trigger events that happen during game play. The game scripts can also be integrated with HTML5 game or animation libraries to create more compelling effects.
  • CNDLS provides the framework code that assembles the Game Design Document and the custom scripts and styles to generate the game experience and to record student interactions.
